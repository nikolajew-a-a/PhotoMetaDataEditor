# Клик по маркеру карты открывает фото в библиотеке

## Why

На экране «Карта» маркеры фотографий сейчас некликабельны — увидев точку в
неожиданном месте, пользователь не может понять, какое это фото, и вынужден
искать его в библиотеке вручную. Это известная недоделка Фазы 4.

## What Changes

- Маркеры на экране «Карта» становятся кликабельными.
- Клик по маркеру переключает приложение на экран «Библиотека» и выбирает
  соответствующее фото (detail-панель справа показывает его превью и метаданные).
- При наведении на маркер — увеличение маркера (визуальная обратная связь,
  если реализуемо без усложнения TileMapView).

## Не входит в объём (non-goals)

- Кластеризация маркеров при скоплении точек.
- Превью фото в тултипе на карте.
- Прокрутка сетки библиотеки к выбранной плитке (желательно, но допустимо
  отложить, если LazyGrid API усложнит задачу — зафиксировать TODO).

## Capabilities

### New Capabilities

- `map-overview`: поведение экрана «Карта» — отображение маркеров всех
  геотегированных файлов и переход к фото по клику на маркер. (Спека новая,
  потому что каталог openspec/specs пока пуст — это первое задокументированное
  поведение экрана.)

### Modified Capabilities

_нет — существующих спек ещё нет_

## Impact

- `:composeApp` — единственный затрагиваемый модуль:
  - `TileMapView` (`ui/map/TileMapView.kt`): колбэк клика по маркеру
    (`onMarkerClick(id)`), хит-тест точки.
  - `MapOverviewComponent`: новый выход «фото выбрано» — колбэк в конструктор.
  - `RootComponent`: связывает выход Map-компонента с Library — переключение
    стека на Library + передача выбранного id.
  - `LibraryComponent`: публичный метод выбора фото по id (`selectPhoto(id)`).
- `:domain` и `:data` не затрагиваются — данные уже есть (Photo.location).
